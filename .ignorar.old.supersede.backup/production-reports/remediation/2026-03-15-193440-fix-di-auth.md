# Fix Report: Hex #5 — OpenFGA DI Singleton

**Date:** 2026-03-15
**Issue:** Three separate `OpenFGAAdapter` instances created (one per port factory)
**Fix:** Added `@lru_cache(maxsize=1)` to `create_authorization_adapter()`

## Files Modified

- `src/siopv/infrastructure/di/authorization.py` — Added `@lru_cache(maxsize=1)` decorator to `create_authorization_adapter()` and updated docstring to explain the singleton intent.
- `tests/unit/infrastructure/di/test_authorization_di.py` — Updated tests:
  - Added `clear_authorization_caches` autouse fixture to clear all 4 caches before each test
  - Changed `test_multiple_calls_create_separate_instances` → `test_multiple_calls_return_same_cached_instance` (asserts `adapter1 is adapter2`)
  - Changed `test_all_ports_from_same_settings_return_different_instances` → `test_all_ports_from_same_settings_return_same_instance` (asserts all 3 ports return the same adapter object)

## Root Cause

Before the fix, each port getter had its own `@lru_cache(maxsize=1)` but all called the uncached `create_authorization_adapter()`. Result: 3 separate adapter instances. After the fix, all port getters delegate to the same cached adapter singleton.
