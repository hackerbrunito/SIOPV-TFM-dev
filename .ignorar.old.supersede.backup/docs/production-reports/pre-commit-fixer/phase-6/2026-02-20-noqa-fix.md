## Task

Fix ruff PLW0108 violation on line 54 with a `# noqa` comment, without replacing the lambda (which would break lru_cache tests).

---

## Root Cause Analysis

`PLW0108` is a **preview rule** in ruff. It does not fire without `--preview` mode. The project's ruff config (`pyproject.toml`) does not enable preview mode.

When `# noqa: PLW0108` was added:
1. ruff's `RUF100` rule (detect unused noqa directives) ran via `--fix`
2. Since `PLW0108` was inactive, the noqa was classified as "unused"
3. `RUF100` stripped the noqa comment
4. Pre-commit failed: "files were modified by this hook"

---

## Fix Applied

**Line 54 (before):**
```python
    monkeypatch.setattr(Settings, "__hash__", lambda self: id(self))
```

**Line 54 (after):**
```python
    monkeypatch.setattr(Settings, "__hash__", lambda self: id(self))  # noqa: PLW0108, RUF100
```

**Why `RUF100` was added to the noqa:** Suppressing `RUF100` on this specific line prevents ruff from removing the `PLW0108` suppression, even when `--preview` is not active. This makes the noqa persistent across ruff `--fix` runs.
