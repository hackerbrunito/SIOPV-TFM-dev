## Design Decisions

### Mocking Strategy
All three adapters use external I/O (Anthropic API, Presidio engines). Tests use:
- `unittest.mock.patch` to intercept `anthropic.Anthropic` constructor
- `AsyncMock` for `asyncio.get_event_loop().run_in_executor` to simulate thread-pool calls
- Module-level attribute patching (`mod.AnalyzerEngine = None`) to test import-error paths
- Direct attribute injection (`adapter._haiku_validator = mock`) for integration tests

### Test Naming Convention
All tests follow `test_{function}_{scenario}` convention:
- `test_init_creates_client_with_api_key`
- `test_empty_text_returns_safe_without_presidio`
- `test_api_exception_returns_true`

### Edge Cases Covered
1. Empty string, whitespace-only, None descriptions
2. Fail-open on ConnectionError, TimeoutError, ValueError
3. Short text (< 20 chars) with and without detections
4. Long text truncation path
5. Missing TextBlock in API response
6. Empty API key disables semantic validation
7. Multiple entity types in single Presidio pass
8. Ordering preservation in multi-vuln execution
