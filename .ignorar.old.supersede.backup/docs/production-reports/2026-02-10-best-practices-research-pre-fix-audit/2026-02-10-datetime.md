# Python Datetime Timezone Best Practices 2026

**Date:** 2026-02-10
**Research:** Current best practices for Python datetime timezone handling

---

## DTZ007 Ruff Rule

**What it catches:** `datetime.strptime()` without `%z` directive (returns naive datetime)

**Fix:** Follow with `.replace(tzinfo=<timezone>)` or `.astimezone()`

**Example:**
```python
# ❌ DTZ007 violation (naive datetime)
dt = datetime.strptime("2026-02-10", "%Y-%m-%d")

# ✅ Fixed
dt = datetime.strptime("2026-02-10", "%Y-%m-%d").replace(tzinfo=timezone.utc)
```

## datetime.now() Variants

**Deprecated (Python 3.12+):**
- `datetime.utcnow()` - Returns naive UTC datetime (misleading, avoid)

**Current Best Practices:**

```python
# ❌ AVOID: Naive datetime in local timezone
datetime.now()

# ✅ RECOMMENDED: Timezone-aware UTC (Python 3.11+)
datetime.now(datetime.UTC)
datetime.now(datetime.timezone.utc)  # Also valid

# ✅ RECOMMENDED: Timezone-aware local time
datetime.now(datetime.timezone.utc).astimezone()
```

**Key Point:** `datetime.now()` without timezone returns naive datetime, causing timestamp conversion issues.

---

## When Naive Datetimes Are Acceptable

**Acceptable:**
- Parsing date-only strings where time/timezone is irrelevant
- Internal calculations where timezone doesn't affect logic
- Known single-timezone context (e.g., logs for local system time)

**NOT Acceptable:**
- Storing in database (use UTC)
- API responses/requests
- Comparing/subtracting across different sources
- Any cross-timezone operations

**Golden Rule:** If in doubt, use timezone-aware.

## Best Import Pattern (Python 3.11+)

```python
# ✅ Modern (Python 3.11+)
from datetime import UTC, datetime, timezone

dt = datetime.now(UTC)  # Clean syntax

# ✅ Compatible (Python 3.9+)
from datetime import datetime, timezone

dt = datetime.now(timezone.utc)
```

**Note:** `datetime.UTC` added in Python 3.11 as alias for `datetime.timezone.utc`.

## strftime/strptime Timezone Handling

**strftime:** Timezone-aware objects include timezone info in output

```python
dt = datetime.now(UTC)
dt.strftime("%Y-%m-%d %H:%M:%S %z")  # Includes +0000
```

**strptime:** Requires `%z` directive to parse timezone

```python
# ✅ With timezone
datetime.strptime("2026-02-10 10:30:00 +0000", "%Y-%m-%d %H:%M:%S %z")

# ❌ Without %z (DTZ007 violation)
datetime.strptime("2026-02-10 10:30:00", "%Y-%m-%d %H:%M:%S")  # Naive
```

---

## Storage Best Practice

**Always store in UTC, convert for display:**

```python
# Store
utc_time = datetime.now(UTC)
db.save(utc_time)

# Display (convert to user's timezone)
user_tz = ZoneInfo("America/New_York")
local_time = utc_time.astimezone(user_tz)
```
