3. **Checkpoint system:** `.build/checkpoints/pending/` tracks unverified Python files. `post-code.sh` creates markers; `/verify` clears them. `pre-git-commit.sh` blocks commits if markers exist.

6. **Permission modes by role:** Read-only auditors use `plan` (no writes). Writers (code-implementer, test-generator) use `acceptEdits`. Runtime agents (smoke-test-runner, dependency-scanner, regression-guard) use `default`.

7. **Naming convention for reports:** Timestamp-based UUID format `{YYYY-MM-DD-HHmmss}-phase-{N}-{agent}-{slug}.md` in `.ignorar/production-reports/{agent}/phase-{N}/`.
