## 1. Hook System

### 1.1 Hook Types Overview Table

| Event | When it fires | Can block? | Matcher support |
|-------|--------------|------------|-----------------|
| `SessionStart` | Session begins or resumes | No | `startup`, `resume`, `clear`, `compact` |
| `UserPromptSubmit` | User submits prompt, before Claude processes | Yes (exit 2) | None (always fires) |
| `PreToolUse` | Before a tool call executes | Yes (exit 2 or hookSpecificOutput) | tool name |
| `PermissionRequest` | When permission dialog appears | Yes (hookSpecificOutput) | tool name |
| `PostToolUse` | After a tool call succeeds | No (shows stderr to Claude) | tool name |
| `PostToolUseFailure` | After a tool call fails | No (shows stderr to Claude) | tool name |
| `Notification` | When Claude Code sends a notification | No | `permission_prompt`, `idle_prompt`, `auth_success`, `elicitation_dialog` |
| `SubagentStart` | When a subagent is spawned | No | agent type |
| `SubagentStop` | When a subagent finishes | Yes (exit 2) | agent type |
| `Stop` | When Claude finishes responding | Yes (exit 2) | None |
| `TeammateIdle` | When an agent team teammate is about to go idle | Yes (exit 2) | None |
| `TaskCompleted` | When a task is being marked as completed | Yes (exit 2) | None |
| `InstructionsLoaded` | When CLAUDE.md or .claude/rules/*.md is loaded | No | None (always fires) |
| `ConfigChange` | When a config file changes during session | Yes (except policy_settings) | `user_settings`, `project_settings`, `local_settings`, `policy_settings`, `skills` |
| `WorktreeCreate` | When worktree is being created | Yes (non-zero exit fails) | None |
| `WorktreeRemove` | When worktree is being removed | No | None |
| `PreCompact` | Before context compaction | No | `manual`, `auto` |
| `SessionEnd` | When session terminates | No | `clear`, `logout`, `prompt_input_exit`, `bypass_permissions_disabled`, `other` |

**Key finding:** `TeammateIdle` and `TaskCompleted` ARE real events (confirmed 2026). Not hypothetical.

### 1.2 PreToolUse

**When:** Before any tool call executes. Fired inside the agentic loop.
**Can block:** Yes — exit code 2 blocks the tool call; stderr is fed to Claude as error message.
**Matcher:** Regex against `tool_name` (e.g., `Bash`, `Edit|Write`, `mcp__.*`)
**Timeout default:** 600 seconds (command hook)

**stdin input:**
```json
{
  "session_id": "abc123",
  "transcript_path": "/path/to/transcript.jsonl",
  "cwd": "/project",
  "permission_mode": "default",
  "hook_event_name": "PreToolUse",
  "tool_name": "Bash",
  "tool_input": { "command": "npm test" }
}
```

**Decision control (hookSpecificOutput):**
```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",          // "allow" | "deny" | "ask"
    "permissionDecisionReason": "reason"
  }
}
```
Can also modify tool input before it runs or inject additional context for Claude.

---

### 1.3 PostToolUse

**When:** After a tool call **succeeds**. Tool already ran — cannot be undone.
**Can block:** No. Exit 2 shows stderr to Claude (does not block anything).
**Matcher:** Regex against `tool_name`

**stdin input** (adds `tool_response` vs PreToolUse):
```json
{
  "session_id": "abc123",
  "hook_event_name": "PostToolUse",
  "tool_name": "Bash",
  "tool_input": { "command": "npm test" },
  "tool_response": { "stdout": "...", "exit_code": 0 }
}
```

**Decision control:** Top-level `decision: "block"` with `reason`. Used to stop Claude from acting on the result.

---

### 1.4 PreCompact / PostCompact

**PreCompact:**
- **When:** Before context compaction (auto or manual)
- **Can block:** No (exit 2 shows stderr to user only)
- **Matcher:** `manual` or `auto`
- **Use case:** Preserve information before summarization deletes it

**PostCompact:** NOT listed in official docs as a separate event. No `PostCompact` event exists in current schema. The lifecycle shows `SessionStart` with matcher `compact` fires after compaction completes — use that instead.

**PreCompact stdin:**
```json
{
  "session_id": "abc123",
  "hook_event_name": "PreCompact",
  "transcript_path": "/path/to/transcript.jsonl"
}
```

---

### 1.5 SessionStart / SessionEnd

**SessionStart:**
- **When:** Session begins (new, resumed, after /clear, after compact)
- **Can block:** No (exit 2 shows stderr to user only)
- **Only `type: "command"` supported** (no HTTP/prompt/agent hooks)
- **Matcher values:** `startup`, `resume`, `clear`, `compact`
- **stdout behavior:** Added as context Claude can see
- **additionalContext:** Can return JSON with `hookSpecificOutput.additionalContext`
- **CLAUDE_ENV_FILE:** Available to set env vars for subsequent Bash commands (append with `>>`)

**SessionEnd:**
- **When:** Session terminates
- **Can block:** No
- **Matcher values:** `clear`, `logout`, `prompt_input_exit`, `bypass_permissions_disabled`, `other`

---

### 1.6 Notification

**When:** Claude Code sends a notification to the user.
**Can block:** No (exit 2 shows stderr to user only — not Claude)
**Matcher:** `permission_prompt` | `idle_prompt` | `auth_success` | `elicitation_dialog`
**Use case:** Custom notification routing (e.g., Slack/sound alerts on idle)

---

### 1.7 TeammateIdle / TaskCompleted

**TeammateIdle:**
- **CONFIRMED REAL** (2026 docs, agent teams feature)
- **When:** Agent team teammate is about to go idle
- **Can block:** Yes — exit 2 prevents teammate from going idle (teammate continues working)
- **JSON `continue: false`** also stops the teammate entirely (same as Stop hook behavior)
- **Matcher:** None (always fires)

**TaskCompleted:**
- **CONFIRMED REAL** (2026 docs)
- **When:** A task is being marked as completed
- **Can block:** Yes — exit 2 prevents the task from being marked completed
- **Matcher:** None (always fires)

### 1.8 Hook Script Formats

**Four handler types:**

1. **`type: "command"`** — Shell script. Receives JSON via stdin, returns via exit code + stdout.
   ```json
   { "type": "command", "command": ".claude/hooks/my-hook.sh", "async": false, "timeout": 600 }
   ```

2. **`type: "http"`** — HTTP POST to URL. JSON as request body, response body as output.
   ```json
   { "type": "http", "url": "http://localhost:8080/hook", "headers": {"Authorization": "Bearer $TOKEN"}, "allowedEnvVars": ["TOKEN"], "timeout": 30 }
   ```

3. **`type: "prompt"`** — Single-turn Claude model evaluation. Returns yes/no JSON decision.
   ```json
   { "type": "prompt", "prompt": "Should this command run? $ARGUMENTS", "model": "claude-haiku-4-5" }
   ```

4. **`type: "agent"`** — Spawns subagent with tools (Read, Grep, Glob) to verify conditions.
   ```json
   { "type": "agent", "prompt": "Verify security policy...", "timeout": 60 }
   ```

**Common fields for all types:** `type`, `timeout`, `statusMessage`, `once` (skills only)

**`async: true`:** Runs command in background without blocking Claude (command hooks only).

**Exit code semantics:**
- Exit 0 → success, parse stdout for JSON
- Exit 2 → blocking error, ignore stdout, feed stderr to Claude
- Any other → non-blocking error, stderr shown in verbose mode (Ctrl+O)

**Settings.json registration format:**
```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          { "type": "command", "command": ".claude/hooks/validate.sh" }
        ]
      }
    ]
  }
}
```

**Skill/agent frontmatter format:**
```yaml
hooks:
  PreToolUse:
    - matcher: "Bash"
      hooks:
        - type: command
          command: "./scripts/security-check.sh"
```

**Path variables:** Use `$CLAUDE_PROJECT_DIR` and `${CLAUDE_PLUGIN_ROOT}` for portable paths.

**All matching hooks run in parallel.** Identical handlers deduplicated automatically.

### 1.9 Known Bugs

**SessionStart compact matcher bug (#15174 / #13650):**
- The `compact` matcher on `SessionStart` may not reliably fire after auto-compaction in some versions.
- Mitigation: Use `PreCompact` to save state before compaction rather than relying on `SessionStart` with `compact` matcher.

**PreCompact transcript_path null bug (#13668):**
- In some versions, `transcript_path` in `PreCompact` stdin may be null/empty.
- Mitigation: Always check `if [ -n "$TRANSCRIPT_PATH" ]` before reading it.

**Hook changes mid-session:**
- Direct edits to hook config files do NOT take effect mid-session (by design, security feature).
- Claude Code captures hook snapshot at startup. External modifications trigger a warning requiring `/hooks` menu review.

**HTTP hooks cannot block via status codes:**
- Only way to block via HTTP hook is to return 2xx with JSON `decision: "block"` or `permissionDecision: "deny"`.
- Non-2xx → non-blocking error (execution continues).

## 2. settings.json Schema

### 2.1 Top-Level Keys (selected most relevant)

| Key | Type | Description |

| `autoMemoryDirectory` | string | Custom auto memory storage location |

| `showTurnDuration` | boolean | Show turn duration messages |

| `otelHeadersHelper` | string | Script for dynamic OTel headers |

### 2.2 Permissions Format

```json
{
  "permissions": {
    "allow": ["Bash(npm run *)", "Read(~/.zshrc)", "WebFetch(domain:example.com)"],
    "ask":   ["Bash(git push *)", "WebFetch(*)"],
    "deny":  ["Bash(curl *)", "Read(./.env)", "Read(./secrets/**)"],
    "additionalDirectories": ["../docs/", "../../shared-utils/"],
    "defaultMode": "acceptEdits"
  }
}
```

**Evaluation order:** deny → ask → allow (deny wins on first match)

**Rule syntax:** `Tool` or `Tool(specifier)` — supports glob patterns, `domain:` prefix for WebFetch

---

### 2.3 Hook Registration

```json
{
  "hooks": {
    "EventName": [
      {
        "matcher": "regex-or-empty",
        "hooks": [
          { "type": "command", "command": "path/to/script.sh", "timeout": 30 }
        ]
      }
    ]
  }
}
```

Three-level nesting: event name → matcher group → handler(s).
Omit `matcher` or use `"*"` / `""` to match all occurrences.

### 2.4 Environment Variables

```json
{ "env": { "NODE_ENV": "production", "CUSTOM_VAR": "value" } }
```

| Variable | Purpose |

### 2.6 Project vs User Level

| Scope | Location | Shareable | Priority |
|-------|----------|-----------|---------|
| Managed | `/Library/Application Support/ClaudeCode/managed-settings.json` (macOS) | Admin | Highest (cannot override) |
| Command Line | CLI flags | — | Temporary only |
| Local | `.claude/settings.local.json` | No (gitignored) | Overrides project & user |
| Project | `.claude/settings.json` | Yes (committed) | Overrides user |
| User | `~/.claude/settings.json` | No | Lowest |

**Array settings MERGE across scopes** (not replace). Example: `permissions.allow` from user + project both apply.

## Summary

**17 hook event types** confirmed in current docs (up from ~6 in early releases). Key additions relevant to SIOPV:
- `TeammateIdle` + `TaskCompleted` → agent team lifecycle hooks (confirmed real)
- `PreCompact` → use for handoff state preservation (but `transcript_path` may be null — bug #13668)
- `SessionStart` matcher `compact` → fires after compaction (but bug #15174 may affect reliability)
- `InstructionsLoaded` → audit when CLAUDE.md files load

**Settings schema** is substantially larger than documented in older guides — 50+ top-level keys in current version. `permissions` uses deny→ask→allow evaluation order. Array settings merge across scopes rather than replace.
