# Round 1 Research: Agent Definitions & Memory System

**Source:** Official Claude Code docs at code.claude.com (fetched 2026-03-13)
**Scope:** Sub-agents format + Memory system — verified online, not from training data

---

## 1. Agent Definition Format

### 1.1 Frontmatter Fields

All supported YAML frontmatter fields for subagent `.md` files:

| Field            | Required | Description |
|:-----------------|:---------|:------------|
| `name`           | Yes      | Unique identifier — lowercase letters and hyphens |
| `description`    | Yes      | When Claude should delegate to this subagent (used for auto-delegation) |
| `tools`          | No       | Allowlist of tools. Inherits all tools if omitted. Supports `Agent(type)` syntax |
| `disallowedTools`| No       | Denylist — tools to remove from inherited or specified list |
| `model`          | No       | Model alias, full model ID, or `inherit`. Defaults to `inherit` |
| `permissionMode` | No       | `default`, `acceptEdits`, `dontAsk`, `bypassPermissions`, or `plan` |
| `maxTurns`       | No       | Maximum agentic turns before subagent stops |
| `skills`         | No       | Skills to inject into subagent context at startup (full content, not invocable) |
| `mcpServers`     | No       | MCP servers scoped to this subagent (inline definitions or string references) |
| `hooks`          | No       | Lifecycle hooks (`PreToolUse`, `PostToolUse`, `Stop`) scoped to this subagent |
| `memory`         | No       | Persistent memory scope: `user`, `project`, or `local` |
| `background`     | No       | `true` = always run as background task. Default: `false` |
| `isolation`      | No       | `worktree` = run in temporary git worktree (auto-cleaned if no changes) |

### 1.2 Required vs Optional

- **Required**: `name` and `description` only
- **All other fields are optional**
- Minimal valid agent definition example:

```markdown
---
name: my-agent
description: Does X when Y
---
System prompt here.
```

### 1.3 Valid Model Values

| Value | Meaning |
|:------|:--------|
| `sonnet` | Alias → latest Sonnet model |
| `opus` | Alias → latest Opus model |
| `haiku` | Alias → latest Haiku model (fastest/cheapest) |
| `claude-sonnet-4-6` | Full model ID (exact version) |
| `claude-opus-4-6` | Full model ID (exact version) |
| `inherit` | Same model as parent conversation |
| (omitted) | Defaults to `inherit` |

### 1.4 Body Format

- Plain Markdown after the closing `---` frontmatter delimiter
- Body becomes the **system prompt** for the subagent
- Subagents receive ONLY this system prompt + basic env details (working dir)
- They do NOT inherit the full Claude Code system prompt
- No maximum length specified in docs, but concise focused prompts work best
- When using `--agents` CLI flag, use `prompt` key for the body content

### 1.5 Discovery and Loading

**Storage locations (priority order, highest first):**
1. `--agents` CLI flag (JSON, current session only, not persisted to disk)
2. `.claude/agents/` (project-level, check into version control for team sharing)
3. `~/.claude/agents/` (user-level, available in all projects)
4. Plugin `agents/` directory (lowest priority)

**When names conflict**: higher-priority location wins.

**Loading timing**: subagents load at session start. Adding a file mid-session requires `/agents` command or session restart to take effect.

**Auto-delegation**: Claude uses each subagent's `description` to decide when to delegate. Phrases like "use proactively" in description encourage automatic delegation.

**Subagents CANNOT spawn other subagents** (no nesting). Nested delegation requires Skills or chaining from main conversation.

### 1.6 New Features (2026)

Features confirmed as new or recently added (Claude Code v2.1.x range):

- **`background` field**: set `true` to always run as background task
- **`isolation: worktree`**: run in temporary git worktree for isolated repo copy
- **`skills` field**: preload full skill content into subagent context at startup
- **`memory` field**: persistent cross-session memory for subagents (`user`/`project`/`local`)
- **`SubagentStart` / `SubagentStop` hook events** in `settings.json`
- **`/agents` command**: interactive UI for managing subagents (create, edit, delete)
- **Resume subagents**: Claude receives agent ID after completion; transcripts at `~/.claude/projects/{project}/{sessionId}/subagents/agent-{agentId}.jsonl`
- **Auto-compaction**: subagents support auto-compaction (triggers at ~95% capacity by default)
- **Agent(type) tool syntax**: restrict which subagent types a coordinator can spawn
- **Task tool renamed to Agent** (v2.1.63): `Task(...)` still works as alias

## 2. Memory System

### 2.1 Memory Types

Claude Code has two complementary memory systems:

| System | Author | Content | Scope | Loaded |
|:-------|:-------|:--------|:------|:-------|
| **CLAUDE.md files** | You | Instructions and rules | Project / user / org | Every session (full) |
| **Auto memory** | Claude | Learnings and patterns | Per working tree (git repo) | Every session (first 200 lines of MEMORY.md) |

**Subagent memory** (third type): subagents can maintain their own persistent memory directory via the `memory` frontmatter field — separate from main auto memory.

### 2.2 MEMORY.md Format

- Plain Markdown file — no required frontmatter
- Lives at `~/.claude/projects/<encoded-project-path>/memory/MEMORY.md`
- Acts as an **index** pointing to topic files (e.g., `debugging.md`, `api-conventions.md`)
- Topic files in same directory contain detailed notes — read on demand, NOT at startup
- Claude writes/updates MEMORY.md and topic files during sessions

**Directory structure:**
```
~/.claude/projects/<project>/memory/
├── MEMORY.md          # Concise index — loaded every session (first 200 lines)
├── debugging.md       # Detailed debug patterns — on-demand only
├── api-conventions.md # API decisions — on-demand only
└── ...                # Any topic files Claude creates
```

### 2.3 Auto-Memory Behavior

- **On by default** (requires Claude Code v2.1.59+)
- Claude decides what's worth saving — not every session
- Saves: build commands, debugging insights, architecture notes, code style preferences, workflow habits
- Triggered by user corrections: "always use pnpm", "remember that API tests require Redis"
- Toggle: `/memory` command → auto memory toggle, OR set `autoMemoryEnabled: false` in settings
- Disable via env: `CLAUDE_CODE_DISABLE_AUTO_MEMORY=1`
- Custom location: `autoMemoryDirectory` setting (accepted in user/local/policy settings; NOT in `.claude/settings.json` — security restriction)

### 2.4 File Structure

**Memory files (topic files) use frontmatter format (as configured in this project):**
```markdown
---
name: memory_name
description: one-line description (used for relevance matching in future sessions)
type: user | feedback | project | reference
---

Memory content here.
For feedback/project types: rule/fact, then **Why:** and **How to apply:** lines.
```

**MEMORY.md itself**: no required frontmatter — acts as a plain index with links to topic files.

### 2.5 Limits and Recommendations

| Item | Limit / Recommendation |
|:-----|:----------------------|
| MEMORY.md lines loaded at startup | First **200 lines** only |
| Content beyond 200 lines | NOT loaded at session start |
| CLAUDE.md size recommendation | Target **under 200 lines** per file |
| CLAUDE.md over limit behavior | Loaded in full (no hard cut), but reduces adherence |
| Topic files | NOT loaded at startup — read on demand |
| Auto memory location | Machine-local only, not shared across machines or cloud |
| Git repo sharing | All worktrees/subdirectories of same git repo share one auto memory dir |

**Key difference**: the 200-line limit applies to MEMORY.md (hard cut at load time). CLAUDE.md files are loaded in full, but longer files reduce adherence quality.

### 2.6 New Features (2026)

- **Auto memory** itself is a new feature (v2.1.59+, early 2026)
- **`autoMemoryDirectory` setting**: customize storage location
- **`autoMemoryEnabled` setting**: project-level toggle
- **`CLAUDE_CODE_DISABLE_AUTO_MEMORY` env var**: session-level disable
- **Subagent persistent memory** (`memory` field in agent frontmatter): `user`, `project`, `local` scopes
  - `user` → `~/.claude/agent-memory/<agent-name>/`
  - `project` → `.claude/agent-memory/<agent-name>/`
  - `local` → `.claude/agent-memory-local/<agent-name>/`
- **`/memory` command**: browse CLAUDE.md files loaded in session, toggle auto memory, open memory folder
- **`InstructionsLoaded` hook**: log which instruction files load and when (useful for debugging)
- **`claudeMdExcludes` setting**: skip specific CLAUDE.md files by path/glob (useful in monorepos)
- **Path-specific rules** in `.claude/rules/`: YAML `paths` frontmatter scopes rules to file patterns
- **Managed policy CLAUDE.md**: org-wide instructions at system path, cannot be excluded by users
