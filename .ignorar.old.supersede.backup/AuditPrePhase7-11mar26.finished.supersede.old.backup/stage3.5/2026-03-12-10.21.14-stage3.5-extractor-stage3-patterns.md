### Streamlit 1.x

- **Verified fact:** Polling must use `@st.fragment(run_every="15s")` — reruns only the fragment, not full page
- **Correct pattern:** `@st.fragment(run_every="15s")` decorator on polling functions
- **Wrong pattern to avoid:** `time.sleep() + st.rerun()` — blocks the sync thread

- **Verified fact:** Port is controlled by `STREAMLIT_SERVER_PORT` env var — `DASHBOARD_PORT` is not natively recognized
- **Correct pattern:** Translate `DASHBOARD_PORT` → `STREAMLIT_SERVER_PORT` in launch script/env
- **Wrong pattern to avoid:** Expecting Streamlit to read `DASHBOARD_PORT` directly

- **Verified fact:** Auth gate uses `st.context.headers.get("Authorization")` for bearer token; OIDC via `st.login("keycloak")` + `.streamlit/secrets.toml`
- **Correct pattern:**
  ```python
  token = st.context.headers.get("Authorization")
  # OR
  st.login("keycloak")  # with .streamlit/secrets.toml configured
  ```
- **Wrong pattern:** No wrong pattern stated — open question remains (see Section 4)

- **Verified fact:** Session state must track `stage`, `current_thread_id`, `decision`; call `st.rerun()` on transitions
- **Correct pattern:**
  ```python
  st.session_state["stage"] = "reviewing"
  st.session_state["current_thread_id"] = thread_id
  st.rerun()
  ```

- **Verified fact:** `st.cache_resource` required for async bridge (Phase 7 prerequisite)
- **Correct pattern:**
  ```python
  @st.cache_resource
  def get_authz_port():
      port = get_authorization_port(get_settings())
      asyncio.run(port.initialize())
      return port
  ```

### fpdf2 2.8.7

- **Verified fact:** `fname` parameter is NOW REQUIRED in `add_font()` since version 2.7 — breaking change
- **Correct pattern:** `pdf.add_font(fname="path/to/font.ttf")`
- **Wrong pattern to avoid:** `pdf.add_font()` without `fname` — raises error in ≥2.7

- **Verified fact:** `open()` and `close()` methods were removed from fpdf2
- **Correct pattern:** Use context managers or direct API; do not call `.open()` or `.close()`
- **Wrong pattern to avoid:** `pdf.open()` / `pdf.close()` — AttributeError

- **Verified fact:** Table API uses context manager in 2.7+
- **Correct pattern:** `pdf.table(col_widths=(...))` as context manager

- **Verified fact:** TOC insertion
- **Correct pattern:** `pdf.insert_toc_placeholder(toc.render_toc, allow_extra_pages=True)`

- **Verified fact:** Page count alias must be called BEFORE `add_page()`
- **Correct pattern:** `pdf.alias_nb_pages()` before any `add_page()` call; use `{nb}` in footer template
- **Wrong pattern to avoid:** Calling `alias_nb_pages()` after pages have been added

- **Verified fact:** In-memory image handling (no disk write needed)
- **Correct pattern:** `pdf.image(io.BytesIO(image_bytes))`
- **Wrong pattern to avoid:** Writing image to temp file then loading path

- **Requirement pin:** `fpdf2>=2.7.0` in pyproject.toml

### 4.2 Integration Risks Not in MEMORY.md

1. **LIME memory leak risk in production:** If Phase 7 renders LIME explanations in Streamlit without `plt.close(fig)`, memory leak accumulates per user interaction. This is a runtime production risk, not a dev-time error.

2. **LangSmith CoT → PDF dependency chain:** Phase 8 PDF generation depends on LangSmith `list_runs` being accurate. If `LANGSMITH_TRACING_V2=true` is not set at pipeline invocation time, CoT data won't be available for PDF. This is a configuration dependency that must be validated at Phase 8 startup.

3. **fpdf2 `alias_nb_pages()` ordering:** If any Phase 8 code adds pages before calling `alias_nb_pages()`, page count in footer will be wrong/broken. Must be enforced as the first call in the PDF generation function.

4. **Redis `aclose()` not `close()`:** If existing code uses `client.close()` it will silently fail in newer redis-py — no error, connection leak. Needs explicit grep before Phase 8 builds.

5. **Jira ADF rejection is silent at runtime:** Plain string descriptions will fail at API call time with a 400 response, not at construction time. Integration tests must include ADF validation.
