The simpler `permissions.deny` approach (`.env`, `credentials*`) achieves the primary security goal with documented API.

**Recommendation:** B — Omit the sandbox block; use `permissions.deny` for credential protection. The sandbox sub-field schema is partially unverified. The security goal (deny writes to sensitive files) is achievable with the fully-verified `permissions.deny` pattern alone.
