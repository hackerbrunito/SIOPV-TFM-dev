# Tenacity 8.2+ Retry Decorator Typing Best Practices (2026)

**Research Date:** 2026-02-10
**Library:** Tenacity 8.2+
**Focus:** Retry decorator typing, mypy strict mode compatibility

---

## Key Findings

### 1. Type Stub Availability

**Status:** Tenacity includes built-in type hints (py.typed marker).
- No separate stubs package required
- Type annotations cover all public API since ~v8.0+
- **Issue:** Type stubs had regressions in v8.2.0, fixed in v8.2.1+

### 2. MyPy Compatibility History

**v8.2.0 Regression (CRITICAL):**
- Removal of function overloads caused mypy failures
- Decorated functions flagged as "not callable"
- Error: "Argument 1 has incompatible type" at decorator sites
- **Fix:** v8.2.1 restored overloads (rapid patch)

**v8.2.2 Fix:**
- Corrected `wait` argument annotation
- Now: `typing.Union[wait_base, Callable[["RetryCallState"], Union[float, int]]]`

**Current Status (8.2.2+):** MyPy compatible, no known blocking issues.

---

## Best Practices

### 3. Proper Typing for @retry Decorator

**Recommended Approach (No type: ignore needed):**
```python
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(stop=stop_after_attempt(3), wait=wait_exponential())
async def fetch_data(url: str) -> dict[str, Any]:
    """MyPy recognizes this correctly in Tenacity 8.2.2+"""
    ...
```

**Why this works:**
- Library provides proper overloads for sync/async functions
- Return type preserved through decorator
- No `# type: ignore` required with Tenacity 8.2.2+

### 4. When to Use `# type: ignore`

**Only if using older versions (<8.2.1):**
```python
@retry(...)  # type: ignore[misc]  # tenacity <8.2.1 missing overloads
def legacy_code() -> int:
    ...
```

**With mypy --strict + --disallow-untyped-decorators:**
- Upgrade to Tenacity 8.2.2+ (fixes removed)
- If stuck on 8.2.0: Use `# type: ignore[misc]` with comment explaining version constraint

**Best Practice:** Avoid `# type: ignore` by upgrading library, not by suppressing errors.

### 5. Async Retry Patterns

**Auto-detection (Recommended):**
```python
@retry(stop=stop_after_attempt(3))
async def async_task() -> str:
    """@retry auto-detects async and uses AsyncRetrying"""
    ...
```

**Manual AsyncRetrying (Advanced):**
```python
from tenacity import AsyncRetrying, stop_after_attempt

async def manual_retry():
    async for attempt in AsyncRetrying(stop=stop_after_attempt(3)):
        with attempt:
            return await fetch()
```

**Type Hints for AsyncRetrying:**
- `sleep: Callable[[float], Awaitable[None]]`
- MyPy correctly infers async context manager protocol

---

## Recommendations

1. **Version Constraint:** Pin `tenacity>=8.2.2` to avoid mypy regressions
2. **MyPy Config:** No special config needed with 8.2.2+, works with `--strict`
3. **Type Ignores:** Not needed for properly versioned Tenacity (8.2.2+)
4. **Async:** Use `@retry` decorator directly (auto-detects async functions)
5. **When Stuck:** If version locked to 8.2.0, add `# type: ignore[misc]` with version comment
