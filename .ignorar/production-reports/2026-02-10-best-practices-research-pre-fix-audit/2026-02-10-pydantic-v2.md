## 1. ConfigDict Pattern (Correct Configuration)

**✅ CORRECT - Pydantic v2:**
```python
from pydantic import BaseModel, ConfigDict

class Vulnerability(BaseModel):
    model_config = ConfigDict(
        strict=True,
        frozen=True,
        str_strip_whitespace=True,
        validate_by_name=True,
        validate_by_alias=True,
    )
```

**Key Updates:**
- `model_config` attribute (dict) replaces v1 `class Config` pattern
- `frozen=True` replaces deprecated `allow_mutation=False`
- `validate_by_name` + `validate_by_alias` provide fine-grained validation control

## 2. Field Validators (@field_validator)

**✅ CORRECT - Pydantic v2:**
```python
from pydantic import BaseModel, field_validator

class CVE(BaseModel):
    cve_id: str

    @field_validator("cve_id")
    @classmethod
    def validate_cve_format(cls, v: str) -> str:
        if not v.startswith("CVE-"):
            raise ValueError("CVE ID must start with 'CVE-'")
        return v
```

**Key Features:**
- `@field_validator` replaces deprecated `@validator`
- Requires `@classmethod` decorator (mypy plugin enforces this)
- `mode="before"` for pre-processing, `mode="after"` for post-validation
- NO `each_item` kwarg - validate container items via type annotation

## 3. Mypy Plugin Integration

**Enable in pyproject.toml:**
```toml
[tool.mypy]
plugins = ["pydantic.mypy"]

[tool.pydantic-mypy]
init_forbid_extra = true
init_typed = true
warn_required_dynamic_aliases = true
```

**Benefits:**
- Synthesizes accurate `__init__` signatures
- Marks validators/serializers as classmethods
- Improves field name vs alias detection
- Strict mode compatibility (as of v2.12+)

**Known Issues:**
- `validation_alias` causes mypy errors (use `alias` for now)
- Property decorators + computed fields can conflict
- Only latest mypy version officially supported

## 4. Deprecated Patterns (v1 → v2)

| v1 Pattern | v2 Replacement | Status |
|-----------|----------------|--------|
| `class Config:` | `model_config = ConfigDict()` | Deprecated |
| `@validator` | `@field_validator` | Deprecated (remove in v3) |
| `@root_validator` | `@model_validator` | Deprecated |
| `allow_mutation = False` | `frozen = True` | Removed |
| `each_item=True` | Type annotation validation | Removed |

## 5. Property Decorators with BaseModel

**⚠️ Current Limitations:**
- Combining `@property` with `@computed_field` causes mypy plugin errors
- Mypy reports: "Decorators on top of @Property are not supported"
- Property setters require `validate_assignment=True` in ConfigDict

**Workaround:**
```python
class Model(BaseModel):
    model_config = ConfigDict(validate_assignment=True)

    _internal: str = ""  # Private field

    @property
    def computed(self) -> str:
        return self._internal.upper()
```
