# Typer 0.12+ Type Hints Best Practices (2026)

**Date:** 2026-02-10
**Researcher:** best-practices-researcher
**Task:** Research Typer type hints for mypy strict mode

---

## Summary

Typer 0.12+ has full PEP 561 support and mypy compatibility. Type stubs are included in the package. The main challenge is handling "untyped decorator" errors with `@app.command()` in strict mode.

---

## Key Findings

### 1. PEP 561 Support (Type Stubs Included)

- **Status:** Typer includes inline type annotations (no separate stubs needed)
- **Version:** Type stub issues resolved since Typer 0.2.1
- **Mypy version:** Requires mypy 0.770+ for full compatibility
- **Source:** [GitHub Issue #99](https://github.com/fastapi/typer/issues/99) - closed as resolved

### 2. Mypy Strict Mode Compatibility

**Problem:** `@app.command()` decorator may trigger "untyped decorator" error

**Solution:** Use selective type ignore with error code:

```python
@app.command()  # type: ignore[untyped-decorator]
def my_command(name: str, age: int = typer.Option(...)) -> None:
    """Properly typed command with selective error suppression."""
    typer.echo(f"Hello {name}, age {age}")
```

**Error Code:** `untyped-decorator` (enabled by `--disallow-untyped-decorators`)

### 3. Best Practice Pattern (2026)

```python
import typer
from typing import Annotated

app = typer.Typer()

@app.command()  # type: ignore[untyped-decorator]
def process(
    input_file: Annotated[Path, typer.Argument(help="Input file path")],
    verbose: Annotated[bool, typer.Option("--verbose", "-v")] = False,
) -> None:
    """Type-safe command with modern Annotated syntax."""
    ...
```

**Why this works:**
- Function signature is fully typed (parameters + return)
- `Annotated` provides metadata without breaking type checker
- Selective ignore suppresses decorator error without disabling strict mode

### 4. Alternative: Decorator Typing Wrapper

For projects requiring zero type ignores, wrap Typer decorators:

```python
from typing import Callable, TypeVar, cast

F = TypeVar('F', bound=Callable[..., None])

def typed_command(func: F) -> F:
    """Type-safe wrapper for @app.command()."""
    return cast(F, app.command()(func))

@typed_command
def my_command(x: int) -> None:
    ...
```

**Trade-off:** More boilerplate but zero type ignores.
