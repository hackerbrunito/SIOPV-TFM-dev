## Known ML Naming Conventions (Pre-Context7)

### 1. Uppercase X Convention - ACCEPTED STANDARD

**Pattern:**
```python
X = features  # Feature matrix (2D array)
y = target    # Target vector (1D array)
X_train, X_test, y_train, y_test = train_test_split(X, y)
```

**Rationale:**
- Mathematical notation convention: X represents feature matrix, y represents target
- Universally adopted in ML/data science community
- Used consistently in:
  - scikit-learn official docs
  - Academic papers
  - ML textbooks (Hastie, Tibshirani, Friedman)
  - Kaggle notebooks
  - Production ML codebases

**PEP8 Conflict:**
- PEP8 N806: Variable in function should be lowercase
- **BUT:** PEP8 explicitly allows overriding for domain-specific conventions
- ML community consensus: uppercase X/y is more readable than x/y for matrices

## 2. Ruff Rule Conflicts

### Rules Triggered by ML Code:

| Rule | Description | ML Convention Conflict |
|------|-------------|----------------------|
| N803 | Argument name should be lowercase | `X`, `y` are standard args |
| N806 | Variable should be lowercase | `X_train`, `X_test`, `y_train`, `y_test` |
| N815 | mixedCase variable | `DMatrix` (XGBoost), `BaseEstimator` |

### Recommended Solution: Per-File Ignores

**Pattern (pyproject.toml):**
```toml
[tool.ruff]
# Global settings
line-length = 100

[tool.ruff.lint]
select = ["E", "F", "W", "N", "D"]

[tool.ruff.lint.per-file-ignores]
# ML modules: Allow uppercase variable names for feature matrices
"src/ml/**/*.py" = ["N806", "N803"]
"src/models/**/*.py" = ["N806", "N803"]
"src/training/**/*.py" = ["N806", "N803"]
"tests/ml/**/*.py" = ["N806", "N803"]

# Notebooks (if using nbqa)
"notebooks/**/*.ipynb" = ["N806", "N803", "T201"]  # Also allow print()
```

**When NOT to ignore:**
- Non-ML modules (API, config, utils)
- Variable names unrelated to feature matrices (e.g., random uppercase vars)
- Only apply ignores to files with scikit-learn/XGBoost usage

## 3. Scikit-learn Official Documentation Examples

**FROM MEMORY (needs Context7 validation):**

Scikit-learn docs CONSISTENTLY use:
```python
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier

# Standard pattern
X, y = load_dataset()
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

clf = RandomForestClassifier()
clf.fit(X_train, y_train)
score = clf.score(X_test, y_test)
```

**Never lowercase:** `x`, `y` are NOT used in official examples.

## 4. XGBoost-Specific Conventions

### DMatrix Naming
```python
import xgboost as xgb

# Standard XGBoost pattern
dtrain = xgb.DMatrix(X_train, label=y_train)
dtest = xgb.DMatrix(X_test, label=y_test)

params = {"max_depth": 3, "eta": 0.1}
model = xgb.train(params, dtrain, num_boost_round=100)
```

**Conventions:**
- `DMatrix` class name is mixedCase (library convention)
- `dtrain`, `dtest` are lowercase (derived object names)
- Still uses uppercase `X_train`, `y_train` as inputs


**Pros:**
- Less maintenance
- Covers future ML files

**Cons:**
- Too broad if domain has non-ML code


**Pros:** None
**Cons:** Noisy, scattered, hard to audit
